## Detalhes de Implementação

Referenciar a seção "Interfaces Principais", "Modelos de Dados" e "Endpoints de API" do techspec.md para contratos da API e constantes.

Requisitos funcionais (seção 2 e 3 do PRD):
- A tela deve exibir 4 campos de entrada, cada um aceitando apenas um dígito numérico
- O usuário deve poder colar um PIN de 4 dígitos e preencher os campos automaticamente
- O usuário deve poder apagar e navegar entre campos de forma previsível
- Ao preencher os 4 dígitos, o usuário deve poder confirmar o PIN
- O frontend deve chamar endpoint específico de validação de PIN com e-mail e PIN
- Ao validar com sucesso, o JWT retornado deve ser salvo em sessionStorage
- Após salvar o token, o usuário deve ser redirecionado para /dashboard
- Se o PIN estiver incorreto, expirado ou exceder tentativas, a tela deve exibir feedback claro
- A tela deve oferecer ação para reenviar o PIN
- O reenvio só deve ficar disponível após cooldown de 1 minuto
- Durante o cooldown, a interface deve indicar que o usuário precisa aguardar

Endpoint POST /auth/verify-pin (techspec.md):
```json
Request: { "email": "usuario@empresa.com", "pin": "1234" }
Response 200: { "accessToken": "<jwt>" }
Erros: 401 com "PIN inválido", 401 com "PIN expirado", 429 com "Tentativas esgotadas"
```

authApi (techspec.md):
```js
export async function verifyPin(email, pin) {
  return authFetch('/auth/verify-pin', { email, pin });
}
```

authSession (techspec.md):
```js
export const authSession = {
  getToken: () => sessionStorage.getItem('milatec:accessToken'),
  setToken: token => sessionStorage.setItem('milatec:accessToken', token),
  clear: () => sessionStorage.removeItem('milatec:accessToken'),
};
```

Constantes (techspec.md):
- PIN_LENGTH = 4
- RESEND_COOLDOWN_MS = 60000

Mensagens de erro (techspec.md):
- PIN inválido: `PIN inválido`
- PIN expirado: `PIN expirado`
- Tentativas esgotadas: `Tentativas esgotadas`
- Cooldown ativo: `Aguarde 1 minuto para reenviar o PIN`
