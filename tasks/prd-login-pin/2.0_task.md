## Detalhes de Implementação

Referenciar a seção "Interfaces Principais" e "Endpoints de API" do techspec.md para contratos da API.

Requisitos funcionais (seção 1 do PRD):
- A tela deve exibir apenas o campo de e-mail e o botão para solicitar PIN
- O campo de e-mail deve validar formato antes de enviar a requisição
- Ao submeter e-mail válido, o frontend deve chamar endpoint específico de solicitacao de PIN
- Se o backend retornar sucesso, o usuário deve ir para a tela de PIN
- Se o e-mail não existir ou estiver inativo, deve exibir exatamente: `Email nao cadastrado ou inativo`
- A tela deve manter os logos e estilo atuais do template

Endpoint POST /auth/request-pin (techspec.md):
```json
Request: { "email": "usuario@empresa.com" }
Response 200: { "message": "PIN enviado para o e-mail.", "expiresInSeconds": 150, "resendAvailableInSeconds": 60 }
Erros: 401 com "Email não cadastrado ou inativo", 429 com "Aguarde 1 minuto para reenviar o PIN"
```

authApi (techspec.md):
```js
export async function requestPin(email) {
  return authFetch('/auth/request-pin', { email });
}
```

authSession (techspec.md):
```js
export const authSession = {
  getToken: () => sessionStorage.getItem('milatec:accessToken'),
  setToken: token => sessionStorage.setItem('milatec:accessToken', token),
  clear: () => sessionStorage.removeItem('milatec:accessToken'),
};
```
