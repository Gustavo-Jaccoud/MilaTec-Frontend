## Detalhes de Implementação

Referenciar a seção "Design de Implementação", "Pontos de Integração" e "Arquivos relevantes" do techspec.md para contratos e integrações.

Requisitos funcionais (seção 5 do PRD):
- Rotas administrativas devem exigir token no sessionStorage
- Usuário sem token deve ser redirecionado para /auth/login
- Usuário autenticado que acessar telas de login deve ser redirecionado para /dashboard
- O logout deve remover o token do sessionStorage

authSession (techspec.md):
```js
export const authSession = {
  getToken: () => sessionStorage.getItem('milatec:accessToken'),
  setToken: token => sessionStorage.setItem('milatec:accessToken', token),
  clear: () => sessionStorage.removeItem('milatec:accessToken'),
};
```

React Router (techspec.md):
- Usa Navigate, useLocation e useNavigate para proteger rotas e preservar destino quando aplicável

Implementação sugerida para PrivateRoute:
```jsx
function PrivateRoute({ children }) {
  const token = sessionStorage.getItem('milatec:accessToken');
  const location = useLocation();

  if (!token) {
    return <Navigate to="/auth/login" state={{ from: location }} replace />;
  }

  return children;
}
```

Implementação sugerida para redirecionamento de autenticado em telas de login:
```jsx
// Em /auth/login e /auth/login-pin
if (sessionStorage.getItem('milatec:accessToken')) {
  return <Navigate to="/dashboard" replace />;
}
```
