## Detalhes de Implementação

Referenciar a seção "Interfaces Principais", "Pontos de Integração" e "Endpoints de API" do techspec.md para contratos e configurações.

authApi (techspec.md):
```js
export async function requestPin(email) {
  return authFetch('/auth/request-pin', { email });
}

export async function verifyPin(email, pin) {
  return authFetch('/auth/verify-pin', { email, pin });
}
```

authFetch deve:
- Usar VITE_API_BASE_URL para montar URL completa
- Usar fetch nativo
- Enviar Authorization header quando token disponível
- Tratar erros HTTP (401, 403, 429, 500)
- Parsear resposta JSON
- Propagar mensagens de erro do backend

VITE_API_BASE_URL (techspec.md):
- Define a URL do backend
- Exemplo local: http://localhost:3000
- Deve ser documentado em README ou documento operacional do frontend

Endpoint POST /auth/request-pin (techspec.md):
```json
Request: { "email": "usuario@empresa.com" }
Response 200: { "message": "PIN enviado para o e-mail.", "expiresInSeconds": 150, "resendAvailableInSeconds": 60 }
Erros: 401 com "Email não cadastrado ou inativo", 429 com "Aguarde 1 minuto para reenviar o PIN"
```

Endpoint POST /auth/verify-pin (techspec.md):
```json
Request: { "email": "usuario@empresa.com", "pin": "1234" }
Response 200: { "accessToken": "<jwt>" }
Erros: 401 com "PIN inválido", 401 com "PIN expirado", 429 com "Tentativas esgotadas"
```
