## Visão Geral

<complexity>HIGH</complexity>

Criar testes unitários para todos os endpoints de autenticação e endpoints de negócio principais, implementar mocks para dependências externas (Airtable, etc), configurar Jest/Vitest para execução de testes, definir coverage mínimo de 80% para endpoints críticos, e criar fixtures de dados para testes consistentes.

<requirements>
- Todos os endpoints de autenticação devem ter testes unitários
- Endpoints de negócio principais devem ter testes unitários
- Dependências externas devem ser mockadas
- Coverage mínimo de 80% em endpoints críticos
- Fixtures de dados para testes consistentes
- Testes devem ser executáveis via npm test
</requirements>

- npm test executa todos os testes sem erros
- Testes de autenticação cobrem cenários de sucesso e falha

- Dependências externas são mockadas corretamente
- Coverage >= 80% em endpoints críticos (auth, projects, users)
- Fixtures de dados garantem testes consistentes
- Coverage report é gerado automaticamente
