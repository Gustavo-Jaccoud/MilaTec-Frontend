# Tarefa 1.0: Adicionar API Backend ao Docker Compose

<critical>Ler os arquivos de prd.md e techspec.md desta pasta, se você não ler esses arquivos sua tarefa será invalidada</critical>

## Visão Geral

<complexity>MEDIUM</complexity>

Criar ou atualizar o arquivo docker-compose.yml na raiz do projeto para unificar frontend e backend em um único comando, facilitando desenvolvimento e deployment. Incluir configurações de rede, volumes e variáveis de ambiente para ambos os serviços.

<requirements>
- docker-compose.yml deve iniciar frontend e backend sem erros
- Frontend deve estar acessível em http://localhost:5173
- Backend deve estar acessível em http://localhost:3000
- PostgreSQL deve estar configurado e acessível
- Hot reload deve funcionar para ambos frontend e backend
- Variáveis de ambiente devem estar configuradas corretamente
</requirements>

## Critérios de Sucesso

- `docker-compose up` inicia frontend, backend e postgres sem erros
- Frontend acessível em http://localhost:5173 com hot reload funcional
- Backend acessível em http://localhost:3000 com hot reload funcional
- PostgreSQL acessível na porta 5432
- Alterações no código refletem automaticamente nos containers (hot reload)
- `docker-compose down` para todos os serviços corretamente
- Logs dos serviços são acessíveis via `docker-compose logs -f`
