# Tarefa 2.0: Integrar Endpoints de Autenticação no Frontend

<critical>Ler os arquivos de prd.md e techspec.md desta pasta, se você não ler esses arquivos sua tarefa será invalidada</critical>

## Visão Geral

<complexity>HIGH</complexity>

Criar serviço de API no frontend com interceptor automático para incluir token JWT em todas as requisições ao backend, implementar tratamento de erros 401/403, criar guards de rota para proteger funcionalidades por role, e implementar verificação de permissões baseada no token JWT.

<requirements>
- apiService.js deve incluir automaticamente token JWT de sessionStorage em todas as requisições
- Interceptor deve redirecionar para /auth/login em erro 401
- Interceptor deve redirecionar para /unauthorized em erro 403
- useAuth hook deve extrair e validar token JWT
- ProtectedRoute deve verificar presença de token e role do usuário
- Funcionalidades devem ser protegidas por role (admin, manager, user)
</requirements>

## Critérios de Sucesso

- Requisições ao backend incluem automaticamente token JWT no header Authorization
- Erro 401 redireciona usuário para /auth/login e remove token do sessionStorage
- Erro 403 redireciona usuário para /unauthorized
- useAuth hook extrai corretamente email, role e outros dados do token
- hasPermission verifica corretamente hierarquia de roles
- ProtectedRoute bloqueia acesso a rotas sem token válido
- ProtectedRoute bloqueia acesso a rotas com role insuficiente
- Rotas administrativas só acessíveis por admin
- Documentação atualizada com exemplos de uso
