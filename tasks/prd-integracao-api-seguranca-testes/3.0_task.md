# Tarefa 3.0: Revisar e Reforçar Segurança do Banco de Dados

<critical>Ler os arquivos de prd.md e techspec.md desta pasta, se você não ler esses arquivos sua tarefa será invalidada</critical>

## Visão Geral

<complexity>HIGH</complexity>

Auditar estrutura atual do banco de dados e implementar validações de entrada, sanitização de queries, RBAC (Role-Based Access Control) no backend, remover dados sensíveis de respostas de API, e implementar rate limiting em endpoints sensíveis para evitar vulnerabilidades comuns como SQL injection.

<requirements>
- Todos os endpoints devem ter validações de entrada usando class-validator
- Queries devem usar prepared statements ou ORM para evitar SQL injection
- RBAC deve ser implementado via guards e decorators
- Dados sensíveis (senhas, tokens) não devem ser expostos em respostas
- Rate limiting deve ser configurado em endpoints sensíveis
- Senhas devem estar criptografadas no banco de dados
</requirements>

## Critérios de Sucesso

- Todos os DTOs têm validações de entrada (email, tamanho, formato, etc)
- RolesGuard bloqueia acesso a endpoints sem role suficiente
- Decorator @Roles funciona corretamente em endpoints
- Endpoints administrativos só acessíveis por admin
- Nenhuma query vulnerável a SQL injection encontrada
- Respostas de API não expõem dados sensíveis
- Rate limiting ativo em endpoints sensíveis
- Senhas criptografadas no banco de dados
- Documentação de segurança atualizada
